# 智慧农业大数据低代码分析与辅助决策平台

## 作品简介（约 500 字）

本作品面向中国大学生计算机设计大赛大数据实践赛道，聚焦智慧农业场景中“数据采集分散、分析处理滞后、结果表达不直观、业务决策依赖经验”等实际问题，设计并实现了一套集数据采集、存储管理、低代码编排、可视化展示和辅助决策于一体的智慧农业大数据低代码分析与辅助决策平台。系统以农业物联网传感器数据为基础，围绕温度、湿度、光照、土壤湿度、颗粒物等环境指标开展数据接入与分析，通过 MQTT 接收设备上报信息，使用 SQLite 进行本地数据管理，并基于 Flask 构建后端服务接口。平台不仅支持原始数据查询，还实现了总览分析、趋势聚合、风险告警、状态评估和决策建议等能力，使系统从“展示数据”进一步提升为“理解数据、分析数据、辅助决策”。

作品的核心特色在于将低代码思想应用到智慧农业大数据场景中。用户无需编写复杂业务代码，即可通过工作流编辑器拖拽节点，完成数据读取、数据库查询、分析摘要生成和结果输出；同时可通过大屏编辑器将关键指标、传感器信息和分析结果进行可视化编排，形成适合演示与汇报的页面。为了便于比赛演示与后续继续开发，系统在应用首次打开时会自动生成一套智慧农业演示工程，包括演示工作流和演示大屏，用户可以直接打开、运行和修改，在此基础上继续扩展业务场景。系统还具备演示兜底能力，当本地数据库样本不足或真实设备未接入时，仍可通过平台内置的演示数据完成完整展示。

本作品体现了大数据实践赛中“数据分析、业务智能、辅助决策”的核心要求，形成了“数据采集-数据治理-分析挖掘-可视化表达-结果输出”的完整闭环，具有较好的场景针对性、展示完整度和应用推广价值。

## 作品安装说明

### 1. 运行环境

- Windows 10/11
- Python 3.9 及以上
- 建议使用 Edge 或 Chrome 浏览器
- 若需要接入真实设备，建议预先安装 MQTT Broker，例如 Mosquitto

### 2. 安装依赖

项目依赖已经整理到 [requirements.txt](./requirements.txt) 中，在项目根目录执行：

```bash
pip install -r requirements.txt
```

如果仅演示平台核心功能，没有 MQTT 环境也可以运行，系统仍可使用本地数据或演示数据完成展示。

### 3. 启动方式

在项目根目录执行：

```bash
python Src/app.py
```

启动后可访问：

- 首页：`http://127.0.0.1:5000/`
- 工作流编辑器：`http://127.0.0.1:5000/workflow-editor`
- 大屏编辑器：`http://127.0.0.1:5000/screen-editor`

### 4. 演示工程说明

- 应用首页首次打开时，会自动在本地项目仓库中生成一套“智慧农业演示工程”
- 该演示工程包含演示工作流项目与演示大屏项目，可直接打开继续编辑
- 演示工作流预置了农业总览、风险告警和决策建议相关节点
- 演示大屏可直接展示传感器数据，并绑定演示工作流端口进行内容展示

### 5. MQTT 可选说明

如需接入真实传感器数据，可单独启动 MQTT 监听：

```bash
python Src/start_mqtt_listener.py
```

若本机未安装 MQTT Broker 或未连接真实设备，平台仍可正常进行界面演示、工作流编排与大屏展示。

## 开源代码和组件使用情况

本作品在开发过程中使用了以下开源代码、开源组件或开源技术方案：

### 1. 后端框架与基础组件

- `Flask`
  - 用途：构建 Web 后端服务、页面路由和数据接口
- `SQLite`
  - 用途：存储设备信息、传感器时序数据和状态数据
- `paho-mqtt`
  - 用途：实现 MQTT 协议通信，接收物联网设备上报数据

### 2. 前端实现方式

- 原生 `HTML + CSS + JavaScript`
  - 用途：实现首页、工作流编辑器和大屏编辑器
- 浏览器原生 API
  - 用途：本地项目存储、数据导入导出、页面交互和可视化渲染

### 3. Python 标准库

- `json`
  - 用途：数据序列化与接口交互
- `sqlite3`
  - 用途：数据库访问
- `threading`
  - 用途：线程局部数据库连接与后台监听支持
- `datetime`
  - 用途：时间处理、趋势分析和统计计算
- `os`、`pathlib`
  - 用途：路径管理与运行环境处理

### 4. 使用说明

- 上述开源组件主要用于平台基础能力搭建，不依赖外部闭源商业系统
- 本作品的核心场景设计、低代码流程编排逻辑、农业数据分析逻辑、演示工程方案与可视化页面均由团队自主设计与实现
- 若在正式提交材料中需要更严格的开源合规说明，建议补充组件版本号、官网地址和许可证信息
